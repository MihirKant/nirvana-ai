## 🎯 PROJECT OVERVIEW

**Project Name:** Nirvana AI  
**Tagline:** Your Personal AI Butler - Locally Powered, Infinitely Capable  
**Category:** AI/ML, Automation, Productivity  
**Platform:** Desktop Application (Windows/Linux/Mac)

### What is Nirvana?
Nirvana is an advanced, locally-running AI assistant that combines the power of Large Language Models (LLMs) with practical system tools, web capabilities, and multimodal features. Unlike cloud-based assistants, Nirvana runs entirely on your machine, ensuring privacy and offline functionality.

### Key Differentiators
- ✅ **100% Local & Private** - No data sent to cloud servers
- ✅ **Tool-Augmented AI** - Can actually DO things, not just chat
- ✅ **Voice & Vision** - Multimodal interaction (speech, camera, screen reading)
- ✅ **Modern UI** - Sleek purple-themed web interface with smooth animations
- ✅ **Extensible** - Easy to add new tools and capabilities

### Component Breakdown

#### 1. Frontend (Web Interface)
- **Technology:** HTML5, CSS3, Vanilla JavaScript
- **Framework:** None (lightweight, no dependencies)
- **Design:** Purple/violet theme with glassmorphism
- **Features:**
  - Real-time chat interface
  - Voice input (Web Speech API)
  - Text-to-speech output
  - Mic toggle switch with smooth animations
  - Camera integration
  - Responsive design

#### 2. Backend (Flask Server)
- **Technology:** Python Flask
- **Port:** 5000 (localhost)
- **Endpoints:**
  - `GET /` - Serves the web interface
  - `POST /chat` - Handles user messages and returns AI responses
- **Features:**
  - JSON API
  - Tool output separation for verbatim display
  - Error handling

#### 3. AI Agent (Core Logic)
- **File:** `src/agent.py`
- **Responsibilities:**
  - Conversation management
  - Tool call parsing (JSON extraction)
  - Tool execution orchestration
  - Response formatting with emotions
- **Key Features:**
  - Few-shot learning examples in system prompt
  - Automatic tool chaining
  - Verbatim tool output display

#### 4. LLM Integration
- **File:** `src/llm.py`
- **Model:** Llama 3.1 8B (via Ollama)
- **API:** Ollama Python SDK
- **Features:**
  - Streaming support
  - Local inference (no internet required)
  - Customizable model selection

#### 5. Tools System
- **File:** `src/tools.py`
- **Total Tools:** 17 functional tools
- **Categories:**
  - System & Files (7 tools)
  - Web & Info (4 tools)
  - Apps & Email (2 tools)
  - Security (2 tools)
  - Visual (2 tools)

## 🧠 AI BEHAVIOR & INTELLIGENCE

### System Prompt Design
The AI is instructed to:
1. **Use tools for actions** - Never claim to do something without using a tool
2. **Output JSON for tool calls** - Structured format for reliability
3. **Show verbatim outputs** - Display tool results exactly as-is
4. **Express emotions** - Add personality to responses
5. **Few-shot learning** - Examples in conversation history

### Tool Call Format
```json
{
    "tool": "tool_name",
    "args": {
        "arg_name": "value"
    }
}
```

### Conversation Flow
1. User sends message
2. Agent adds to history
3. LLM generates response
4. If JSON tool call detected:
   - Parse and execute tool
   - Add tool output to history
   - LLM generates final response
5. Return response to user

### Prerequisites
1. **Python 3.8+**
2. **Ollama** - Download from ollama.ai

### Installation Steps

```bash

# 2. Install Python dependencies
```

## 📚 TECHNICAL DOCUMENTATION

### API Endpoint: `/chat`
**Method:** POST  
**Content-Type:** application/json

**Request:**
```json
{
    "message": "What's the weather?"
}
```

**Response:**
```json
{
    "response": "[NEUTRAL] The weather in Mumbai is 28°C, partly cloudy.",
    "tool_outputs": [
        {
            "tool": "get_weather",
            "output": "Mumbai: ⛅️ 28°C"
        }
    ]
}
```

### Adding a New Tool

1. **Define function in `tools.py`:**
```python
def my_new_tool(param1, param2):
    # Implementation
    return "Result"
```

2. **Add to AVAILABLE_TOOLS dict:**
```python
AVAILABLE_TOOLS = {
    # ... existing tools
    "my_new_tool": my_new_tool
}
```

3. **Update system prompt in `agent.py`:**
```python
SYSTEM_PROMPT = """
...
18. my_new_tool(param1: str, param2: str): Description
...
"""
```
